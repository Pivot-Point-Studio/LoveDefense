# 성장 리포트 기능 개발 명세서

## 1. 기능 개요

사용자가 연애 시뮬레이션을 진행하면서 자신의 대화 능력 변화를 확인할 수 있는 **성장 리포트 기능**을 추가한다.

기존 사용자(Profile) 탭에서 스크롤을 내리면 성장 리포트 영역이 나오도록 구성한다.

성장 리포트는 단순 점수 표시가 아니라 사용자가 어떤 부분에서 성장하고 있는지 직관적으로 확인할 수 있는 형태로 제작한다.

---

# 2. 화면 구성

현재 사용자 탭:

```
사용자 탭
 ├─ 내 성향 / 상대 성향 설정
 └─ 저장한 추천 표현
```

변경 후:

```
사용자 탭
 ├─ 내 성향 / 상대 성향 설정
 ├─ 성장 리포트
 │
 │  ├─ 성장 그래프
 │  └─ 맞춤형 연애 조언
 │
 └─ 저장한 추천 표현
```

# 3. 성장 리포트 UI

## 제목

사용자 탭 내부에서 큰 제목으로 표시

```
성장 리포트
```

---

# 4. 성장 그래프

## 목적

사용자가 게임을 진행하면서 자신의 연애 대화 능력이 어떻게 성장했는지 확인할 수 있도록 한다.

기존 게임 평가 시스템에서 사용하는 4가지 지표를 그대로 활용한다.

사용 지표:

- Context
- Emotion
- Communication
- Non-Leap

---

# 5. 그래프 형태

## 기본 형태

꺾은선 그래프(Line Chart) 사용

예:

```
점수

100 |                 ●
 80 |          ●
 60 |     ●
 40 |
 20 | ●
    ---------------------
       1   2   3   4회차
```

---

# 6. 그래프 디자인

현재 서비스 디자인은 흰색 배경 + 인스타그램 계열 컬러를 사용하고 있으므로 기존 디자인과 통일한다.

파스텔톤은 사용하지 않는다.

각 지표는 서로 다른 인스타그램 계열 컬러를 사용한다.

예시:

```
Context
→ 퍼플 계열

Emotion
→ 핑크 계열

Communication
→ 오렌지 계열

Non-Leap
→ 블루 계열
```

정확한 색상은 현재 프로젝트 CSS 컬러와 최대한 통일한다.

---

# 7. 그래프 인터랙션

## 지표 선택 버튼

그래프 위에 4개의 지표 버튼을 배치한다.

예:

```
[Context]
[Emotion]
[Communication]
[Non-Leap]
```

## 동작 방식

한 번에 하나의 그래프만 표시한다.

초기 상태:

```
Context 선택

↓

Context 그래프 표시
```

사용자가 Emotion 버튼 클릭:

```
기존 Context 그래프 제거

↓

Emotion 그래프 표시
```

즉, 선택한 지표의 그래프만 화면에 나타난다.

---

# 8. 그래프 애니메이션

그래프는 단순히 나타나는 방식이 아니라 선이 그려지는 애니메이션 효과를 적용한다.

동작:

```
화면 진입

↓

그래프 시작점 표시

↓

선을 따라 이동하며 그래프 생성

↓

최종 그래프 완성
```

지표 변경 시:

```
기존 그래프 제거

↓

새로운 지표 그래프 선 애니메이션 실행
```

# 9. 성장 데이터 구조

기존 게임 결과 데이터를 활용한다.

현재 저장되는 평가 데이터:

```
context_score
emotion_score
communication_score
non_leap_score
```

활용 예시:

```javascript
{
  date: "2026-07-15",
  context: 70,
  emotion: 80,
  communication: 65,
  nonLeap: 90
}
```

---

# 10. 영역별 성장 배지

성장 리포트 내부에 영역별 성장 배지를 추가한다.

각 지표 점수가 일정 기준 이상이면 배지를 획득한다.

예:

```
Context 80점 이상
→ 상황 이해왕 
```

```
Emotion 80점 이상
→ 공감 마스터 
```

```
Communication 80점 이상
→ 대화 해결사 
```

```
Non-Leap 80점 이상
→ 관계 안정가 
```

배지는 게임 진행 기록을 기반으로 자동 획득한다.

# 11. 맞춤형 연애 조언

## 목적

사용자와 상대방의 성향 데이터를 기반으로 AI가 개인 맞춤형 연애 조언을 제공한다.

# 입력 데이터

사용자 정보:

```
이름
MBTI
연애 기간
성향
```

상대 정보:

```
이름
MBTI
성향
```

예:

```javascript
{
 user:{
   name:"민수",
   mbti:"ENTP",
   relationshipLength:"1년",
   tendency:"장난이 많고 솔직한 편"
 },

 partner:{
   name:"서연",
   mbti:"ISFJ",
   tendency:"감정을 오래 참다가 한번에 표현하는 편"
 }
}
```

---

# 12. AI 조언 출력 형태

긴 분석 형태가 아니라 3~4줄 정도의 짧은 조언으로 제공한다.

예:

```
민수님은 솔직하게 표현하는 장점이 있지만,
상대가 감정을 정리할 시간을 기다려주는 연습이 필요해요.

서연님은 작은 배려를 중요하게 생각하는 성향이라
문제 해결보다 먼저 감정을 확인해주는 것이 도움이 됩니다.
```

---

# 13. RAG 적용 방향 (추후 적용)

맞춤형 연애 조언 영역에 RAG 적용 가능.

구조:

```
사용자 정보 입력

↓

사용자 / 상대 성향 분석

↓

RAG 데이터베이스 검색

↓

비슷한 성향의 연애 패턴 검색

↓

검색 결과를 GPT에게 전달

↓

개인 맞춤형 연애 조언 생성
```

# 14. 구현 위치

현재 프로젝트 구조 기준:

```
src
 ├─ App.jsx
 ├─ components
 ├─ data
 └─ repositories
```

성장 리포트 컴포넌트 추가:

```
src/components/
 └─ GrowthReport.jsx
```

필요하면 그래프 컴포넌트 분리:

```
src/components/
 ├─ GrowthReport.jsx
 ├─ GrowthChart.jsx
 ├─ GrowthBadge.jsx
 └─ RelationshipAdvice.jsx
```

# 16. 구현 범위

이번 구현은 로컬 환경에서 동작하는 것을 목표로 한다.

기존 DB 구조 변경은 하지 않는다.

기존 게임 결과 데이터 또는 임시 Mock 데이터를 활용해서 먼저 UI와 기능을 구현한다.

추후 Supabase 데이터 연결 및 AI API 연결을 진행한다.
