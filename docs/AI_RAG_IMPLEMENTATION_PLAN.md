# 연애 디펜스 AI RAG 적용 계획서

## 1. RAG 도입 목적

현재 연애 디펜스 서비스의 AI 기능은 사용자의 프로필 정보와 현재 상황 정보를 OpenAI에게 전달하여 답변을 생성하는 방식이다.

현재 구조:

```
사용자 정보
- MBTI
- 성향
- 상대방 성향
- 연애 기간

+

현재 갈등 상황

↓

OpenAI

↓

AI 답변 생성
```

하지만 현재 방식은 AI가 일반적인 연애 지식 기반으로 답변하기 때문에,
사용자와 상대방의 성향을 기반으로 한 일관성 있는 개인화 조언을 제공하기 어렵다.

이를 개선하기 위해 RAG(Retrieval Augmented Generation)를 적용한다.

---

# 2. RAG란?

RAG는 AI가 답변을 생성하기 전에 외부 데이터베이스에서 관련 정보를 검색하고,
검색된 정보를 참고하여 답변을 생성하는 방식이다.

기존 방식:

```
사용자 입력

↓

GPT

↓

답변 생성
```

RAG 적용 방식:

```
사용자 입력

↓

관련 데이터 검색

↓

검색 결과(Context)

↓

GPT에게 전달

↓

답변 생성
```

즉, GPT가 가진 일반적인 지식만 사용하는 것이 아니라
서비스에서 직접 구축한 연애 데이터를 참고하여 더욱 개인화된 답변을 생성할 수 있다.

# 3. 현재 서비스 구조와 RAG 적용 위치

현재 서비스 흐름:

```
사용자 입력

↓

사용자/파트너 프로필

↓

현재 상황 정보

↓

OpenAI

↓

AI 대화 생성 및 평가
```

RAG 적용 후:

```
사용자 정보
+
상황 정보

↓

Persona RAG 검색

↓

유사한 연애 패턴 검색

↓

검색 결과(Context)

↓

OpenAI Prompt 추가

↓

개인화된 AI 답변 생성
```

---

# 4. RAG 적용 범위

## 1차 적용 대상

게임 진행 로직이 아닌:

```
성장 리포트
└── 맞춤형 연애 조언
```

부분에만 적용한다.

## 적용 이유

- 현재 게임 평가 로직은 정상적으로 동작 중
- 기존 evaluate-turn 구조 변경 위험 방지
- 성장 리포트의 AI 조언은 개인화 효과가 가장 중요함
- 추후 확장하기 쉬운 구조

---

# 5. Persona RAG 개념

## 목적

미리 다양한 연애 성향 데이터를 구축하고,
사용자가 입력한 성향과 유사한 페르소나를 검색한다.

검색된 페르소나 정보를 AI에게 전달하여
사용자에게 맞는 연애 조언을 생성한다.

---

# 6. Persona 데이터 구조

예시:

```json
{
  "title": "해결책 우선형",
  "description": "문제가 발생하면 감정보다 원인 분석과 해결 방법을 먼저 찾는 유형",
  "relationship_pattern": "상대는 공감 부족으로 느낄 수 있음",
  "advice": "문제 해결 전에 상대 감정을 먼저 확인하는 연습이 필요함"
}
```

---

# 7. Persona 데이터 예시

## 1) 해결책 우선형

특징:

- 문제 발생 시 원인 분석을 먼저 함
- 해결 방법 제시가 빠름
- 상대는 공감 부족으로 느낄 가능성이 있음

추천 조언:

```
해결 방법을 찾기 전에 상대가 어떤 감정을 느꼈는지 먼저 확인하는 과정이 필요합니다.
```

## 2) 감정 확인형

특징:

- 상대의 관심 표현을 중요하게 생각함
- 연락 빈도와 애정 표현에 민감함

추천 조언:

```
상대의 행동보다 그 행동이 나에게 어떤 의미였는지 차분하게 전달하는 연습이 도움이 됩니다.
```

## 3) 감정 누적형

특징:

- 서운한 상황을 바로 표현하지 않음
- 작은 불만이 쌓이다가 한 번에 표현됨

추천 조언:

```
작은 서운함을 쌓아두기보다 부담 없는 방식으로 미리 표현하는 것이 관계 유지에 도움이 됩니다.
```

## 4) 독립 성향형

특징:

- 혼자 문제를 해결하려는 성향
- 상대에게 의존하는 것을 부담스럽게 생각함

추천 조언:

```
혼자 해결하는 능력도 중요하지만 상대에게 도움을 요청하는 것도 관계 표현입니다.
```

# 10. 로컬 RAG 구현 방향

현재 단계에서는 실제 서비스 배포 목적이 아니라
로컬 테스트용 RAG 구현을 목표로 한다.

Supabase 구조 변경 없이 구현한다.

추천 구조:

```
src/backend/rag/

├── personaData.js
├── personaRetriever.js
├── vectorStore.js
└── ragService.js
```

---

# 11. 파일 역할

## personaData.js

미리 정의된 연애 페르소나 데이터 저장

예:

```javascript
[
 {
  title: "논리 해결형",
  description: "감정보다 해결책을 먼저 찾는 유형",
  advice: "공감 표현을 먼저 하는 것이 필요함"
 }
]
```

## personaRetriever.js

사용자의 성향과 가장 유사한 Persona 검색

역할:

```
사용자 정보

↓

Similarity Search

↓

유사 Persona 반환
```

## vectorStore.js

벡터 저장 및 검색 담당

초기:

- 로컬 Vector Store

확장:

- ChromaDB
- Supabase pgvector

## ragService.js

전체 RAG 흐름 관리

역할:

```
사용자 정보 입력

↓

Persona 검색

↓

Context 생성

↓

OpenAI 전달
```

---

# 12. Vector DB 선택

초기 데이터 규모:

- Persona 50~200개
- 상황 데이터 100개 수준

에서는 큰 DB가 필요하지 않다.

초기 구현:

```
Local Vector Store
```

이후 확장:

```
Local Vector Store

↓

ChromaDB

또는

```

# 14. Prompt 전달 예시

```
사용자 성향:
ENTP

특징:
문제가 생기면 해결책을 먼저 제시하는 편


상대방 성향:
감정 표현과 공감을 중요하게 생각함


검색된 Persona 정보:

1. 논리 해결형
- 문제 해결 능력은 좋지만 상대가 공감 부족으로 느낄 수 있음

2. 감정 누적형
- 서운함을 바로 표현하지 않고 쌓아두는 경향


위 정보를 참고하여 사용자의 성장 리포트에 표시할
맞춤형 연애 조언을 3~4줄로 작성하세요.
```

# 16. 기대 효과

기존 AI:

```
상대방 입장에서 생각해보세요.
```

일반적인 조언 제공

RAG 적용 후:

```
최근 대화 기록과 성향을 보면
문제를 해결하려는 능력은 강점이지만,
상대는 해결책보다 먼저 감정을 이해받고 싶어하는 패턴이 있습니다.

대화를 시작할 때 해결 방법보다
"그때 어떤 기분이었어?"라는 질문을 먼저 해보면 좋습니다.
```

개인화된 연애 코칭 가능
